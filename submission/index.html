Project Summary

BeyondChats ingests articles, searches for external references via Google (SerpApi), scrapes reference pages (Puppeteer), and uses an LLM (Google Gemini) to rewrite and enhance article content. The backend is a Node/Express API and the frontend is a React + Vite app.

  • Client: React + Vite
  • Server: Node.js, Express, Mongoose
  • Scraping: SerpApi + Puppeteer
  • LLM: Google Gemini (@google/genai) wrapper (optional)

Architecture Notes

Key components and interactions:

  1. Client calls Express API endpoints under /api/articles.
  2. The controller (articleController.js) handles CRUD and the `update-from-google` flow.
  3. `update-from-google` flow: fetch article → search + scrape → rewrite via LLM → update DB (save updatedContent + references).
  4. MongoDB stores `originalContent`, `updatedContent`, and `references` for each article.
